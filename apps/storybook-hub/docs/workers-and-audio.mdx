# Workers and Audio

The loader and player support worker-based execution paths to reduce main-thread pressure.

Use this page when you want to understand:

- when to use worker rendering/loading
- how worker/worklet options change runtime behavior

## Worker-oriented Player Configuration

```ts
const player = createPlayer({
  options: {
    canvas,
    audio,
    renderMode: 'worker',
    loadTransport: 'worker',
    audioEngineMode: 'worklet',
  },
});
```

## Local-development Diagnostics

Performance HUD and export tooling are intentionally local-development diagnostics and are not part of the standard consumer integration flow or `@cxing/media-player` runtime API.

## Common Mistakes

- Forcing worker mode in environments without required support checks.
- Forgetting to align rendering mode and audio mode with your product requirements.
- Assuming worker mode is always faster without validating in your target browsers.
