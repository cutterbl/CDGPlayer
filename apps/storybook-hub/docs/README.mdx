# Introduction

This documentation is for developers integrating CDG playback into their own app.

Use this as a guided reference map:

- [Getting Started](/?path=/docs/documentation-getting-started--docs): install, run, and build the workspace locally.
- [Migration Guide](/?path=/docs/documentation-migration-guide--docs): move from legacy monolithic packages to the modular `@cxing/media-*` packages.
- [Architecture](/?path=/docs/documentation-architecture--docs): understand how loader, player, and controls fit together.
- [Logger Contract](/?path=/docs/documentation-api-logger-contract--docs): scoped logger API (`log/info/warn/error/debug`) and debug gating behavior.
- [API > Loader Contract](/?path=/docs/documentation-api-loader-contract--docs): input types and normalized loaded track payload.
- [API > Player Contract](/?path=/docs/documentation-api-player-contract--docs): playback lifecycle, state, and runtime events.
- [API > Controls Contract](/?path=/docs/documentation-api-controls-contract--docs): adapter model and UI control behavior.
- [Workers and Audio](/?path=/docs/documentation-workers-and-audio--docs): worker/worklet modes and practical runtime configuration.
- Framework-agnostic implementation guide: open `Framework-agnostic Examples` in the composed sidebar.
- React implementation guide: open `React Examples` in the composed sidebar.
- [Contribution Guidelines](/?path=/docs/documentation-contribution-guidelines--docs): pull request workflow and contribution standards.
- [Overview](/?path=/docs/documentation-overview--docs): quick orientation of the composed docs and example catalogs.
